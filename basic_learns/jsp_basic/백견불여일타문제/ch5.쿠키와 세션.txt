-------- ch5.쿠키와 세션 --------

1.
주된 이유 : 브라우저는 비연결지향이므로, 연결 및 요청 -> 응답 하고 데이터를 담지 않는다. 그래서 클라이언트 단에서 쿠키라는 임시 파일을 만들어서 데이터를 보관
문제점 : 쿠키 폴더를 알면 , 조작이나 그런것이 가능하므로 보안에 문제 있음

==>
이유 : 간단한 정보를 클라이언트에 저장함으로써 서버의 부하를 크게 줄일 수 있음.
문제점 : 보안문제 유발하므로 , 웹 브라우저 자체에 쿠키 거부 기능이 있음.
		-> 웹 브라우저에 쿠키 거부 기능이 설정되어있으면, 브라우저와의 연결을 지속시키는 기능을 수행할 수 X

3.
웹 브라우저는 비 연결 지향이므로, 다른 웹 페이지로 이동할때 로그인 상태와 같은 클라이언트의 정보를 유지할 필요가 있음

==>
HTTP 프로토콜은 웹 브라우저(클라이언트)요청에 대한 응답하고나면 클라이언트와 연결을 지속하지 않음.
그래서 이전 작업이나 현재 작업 중인 것에 대한 정보를 가지고 있지 않음
-> 세션을 활용해서 웹 서버에 웹 브라우저의 정보를 저장해서 이후에 계속되는 웹 브라우저의 요청 시 해당 웹 브라우저의 정보와 서버에 저장되어있는
	각각의 웹 브라우저에 대한 정보를 비교해서 동일한 웹 브라우저로부터 온 요청을 판단할 수 있음. 로그인된 사용자 or 장바구니 등의 서비스를 제공하기 위해
	지속적인 상태 관리가 필요한데, 상태 관리를 위해서 세션을 이용
