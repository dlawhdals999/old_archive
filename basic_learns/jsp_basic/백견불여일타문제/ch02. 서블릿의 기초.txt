-------- ch02. 서블릿의 기초 --------

1.
서블릿 생명 주기 : 초기 호출 -> Servlet 인스턴스 생성 & 요청 처리  // 두번 째 이후 호출 -> 기존 인스턴스로 요청 처리
			   톰캣 종료시 인스턴스 소멸
메소드의 종류와 기능 : 
init() => 초기 서블릿 클래스가 로드될 때 인스턴스 생성하면서 초기화
doGet() => GET요청방식을 처리, URL에 parameter가 나타남
doPost() => POST 요청방식을 처리
destory() => 인스턴스 소멸시 호출되는 메소드(자원정리)
			   
--------------------------------------------------------
2.
doGet => response.setContentType("... charset = utf-8");
doPost => request.setCharacterEncoding("UTF-8");


===>
1)클라이언트에게 전송할 메시지 중 한글 깨짐을 방지하기 위한 방법
response.setContentType("... charset = utf-8");
request.setCharacterEncoding("UTF-8");

2)입력 폼에서 입력된 한글을 깨지지 않도록 하는 방법
get : 서버 설정 파일인 server.xml 파일에서 <Connector> 태그를 찾아서 URIEncoding 속성 추가
post : request.setCharacterEncoding("UTF-8");
